## Key Discovery: Hermes CLI Plugin API

The Hermes Agent plugin system provides two ways to register CLI commands:

### For general plugins
`ctx.register_cli_command(name, help, setup_fn, handler_fn)` on `PluginContext` during `register(ctx)`.

### For memory provider plugins (our case)
A **convention-based approach**: add a `register_cli(subparser)` function to `cli.py`. The memory provider plugin discovery system auto-detects it and wires it into `hermes <subcommand>` — no `ctx.register_cli_command()` call needed. Verified against official Hermes docs at `website/docs/guides/build-a-hermes-plugin.md` and the `plugins/memory/` discovery system.

**Impact:** CLI is self-contained in a single new file (`cli.py`). The existing `register()` in `__init__.py` is untouched.

```python
"""CLI subcommands for the ai-memory Hermes plugin.

Hermes memory provider discovery automatically loads a ``register_cli(subparser)``
function from ``cli.py``.  No ``ctx.register_cli_command()`` call needed.

Subcommands:
  status  —  connection reachability + wiki page/session counts
  config  —  saved configuration from $HERMES_HOME/ai-memory.json
  link    —  symlink plugin dir into $HERMES_HOME/plugins/ai-memory
"""

from __future__ import annotations

import argparse
import json
import os
from pathlib import Path

from client import AiMemoryClient
from config import AiMemoryConfig, load_config


def get_hermes_home() -> str:
    return os.environ.get("HERMES_HOME", os.path.expanduser("~/.hermes"))


def cmd_status(args: argparse.Namespace) -> None:
    """Print connection status and wiki statistics.

    Invoked by ``hermes ai-memory status``.  Displays the server URL,
    reachability (ok / error with detail), and wiki page/session counts
    when reachable.
    """
    cfg = load_config(get_hermes_home())
    client = AiMemoryClient(cfg)
    print(f"Server: {cfg.server_url}")
    try:
        result = client.status()
        pages = result.get("pages", "?")
        sessions = result.get("sessions", "?")
        print("Status: ok")
        print(f"Pages:  {pages}")
        print(f"Sessions: {sessions}")
    except Exception as exc:
        print(f"Status: error \u2014 {exc}")


def cmd_config(args: argparse.Namespace) -> None:
    """Print saved configuration.

    Invoked by ``hermes ai-memory config``.  Reads the JSON config file
    and prints each key:value pair on its own line.  Shows a clear error
    when the file is missing or corrupt.
    """
    path = Path(get_hermes_home()) / "ai-memory.json"
    if not path.exists():
        print(f"Error: no configuration found at {path}")
        return
    try:
        data = json.loads(path.read_text())
    except json.JSONDecodeError:
        print(f"Error: corrupt configuration at {path}")
        return
    for key, value in data.items():
        print(f"{key}: {value}")


def cmd_link(args: argparse.Namespace) -> None:
    """Symlink plugin directory into ``$HERMES_HOME/plugins/ai-memory``.

    Invoked by ``hermes ai-memory link``.  Idempotent: prints a message and
    returns early if the symlink already exists.
    """
    source = Path(__file__).parent.resolve()
    hermes_home = get_hermes_home()
    target = Path(hermes_home) / "plugins" / "ai-memory"
    if target.is_symlink() or target.exists():
        print(f"Already linked: {target}")
        return
    target.parent.mkdir(parents=True, exist_ok=True)
    target.symlink_to(source, target_is_directory=True)
    print(f"Linked: {source} -> {target}")


def register_cli(subparser: argparse._SubParsersAction) -> None:
    """Register ai-memory CLI subcommands with the Hermes CLI.

    Called automatically by the Hermes memory provider plugin discovery
    system when it finds a ``register_cli`` function in ``cli.py``.
    """
    p = subparser.add_parser("status", help="Show connection status and wiki statistics")
    p.set_defaults(handler=cmd_status)

    p = subparser.add_parser("config", help="Show saved ai-memory configuration")
    p.set_defaults(handler=cmd_config)

    p = subparser.add_parser("link", help="Link plugin into $HERMES_HOME/plugins/")
    p.set_defaults(handler=cmd_link)
```

### Test monkeypatching note

The handler functions in `cli.py` import `AiMemoryClient` and `load_config` **inside function bodies** (lazy imports), which is critical for monkeypatching:

```python
def cmd_status(args):
    cfg = load_config(...)          # imported at call time
    client = AiMemoryClient(cfg)   # imported at call time
```

This means `monkeypatch.setattr("cli.AiMemoryClient", ...)` patches the reference **before** the handler body executes. If a future refactor moves these to module-level `from client import AiMemoryClient`, the monkeypatch will silently stop working (Python caches the module reference). **Do not move imports to module level without updating the test patching strategy.**
